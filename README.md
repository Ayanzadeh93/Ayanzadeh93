### 🔬 Research Pipeline

<!-- ─── CUSTOM PIPELINE DIAGRAM (custom SVG, hand-designed for your work) ──── -->
<p align="center">
  <img src="./assets/research-pipeline.svg" alt="research-pipeline" width="100%"/>
</p>

> The pipeline above is the architecture I work on day-to-day. **Perception** extracts rooms and objects from raw floor plans using VLMs and YOLO-based detectors. **Representation** turns that into a navigable spatial graph plus a retrieval store. **Reasoning** uses agentic RAG to plan paths and verify them against hallucinations. The output is a natural-language route a blind or low-vision user can actually follow.

### 🚀 Active Projects

<table>
<tr>
<td width="50%" valign="top">

#### 🗺️ Floorplan2Guide
LLM-guided agentic floor plan parsing for indoor BLV navigation.
- **Venue:** CVPR Workshop 2025
- **Stack:** `Python` `VLMs` `NetworkX` `LangChain`
- **Key idea:** decompose floorplan understanding into agent sub-tasks (rooms, doors, adjacency) for grounded path queries.

</td>
<td width="50%" valign="top">

#### 🤖 AgenticRAG
Multi-agent RAG pipeline for accessible navigation instructions.
- **Status:** Active research
- **Stack:** `LangGraph` `RAG` `Multi-agent` `Qwen2-VL`
- **Key idea:** specialist agents (planner, verifier, instructor) coordinate to suppress LLM hallucination in safety-critical routing.

</td>
</tr>
<tr>
<td width="50%" valign="top">

#### 💬 AidGPT
PhD thesis — conversational assistive agent for BLV users.
- **Status:** In development
- **Stack:** `PyTorch` `Multimodal LLMs` `Speech`
- **Key idea:** real-time multimodal dialogue grounded in physical environment context.

</td>
<td width="50%" valign="top">

#### 🔭 InVLM
LoRA fine-tuning of Qwen2-VL for indoor scene understanding.
- **Status:** Ongoing
- **Stack:** `PEFT` `Qwen2-VL` `🤗 Transformers`
- **Key idea:** parameter-efficient adaptation of VLMs to indoor floor-plan / navigation domains.

</td>
</tr>
